## 1. What this repo is

This is the **Govena website** — a 100% static HTML site deployed on Netlify at `govena.com.au`. It is the customer-facing frontend for Govena's AI Governance Pack product.

**There is no build step. No npm. No framework. No Sass. No bundler.**  
Every page is a standalone `.html` file. All CSS is written inside `<style>` tags in `<head>`. All JavaScript is inline `<script>` tags. This is intentional and must be preserved.

The site handles:
- Marketing and conversion (homepage, pricing, how it works, FAQ)
- A multi-step intake questionnaire customers fill in before purchasing
- A live document preview generated by Claude AI, shown before purchase to demonstrate quality
- Post-payment success page
- A resource hub with 20 SEO-targeted articles on AI governance for Australian SMEs

---

## 2. Relationship to govena-backend

This repo contains **only frontend files**. There is no server-side code here.

All API calls (saving questionnaire answers, creating Stripe checkout sessions, generating document previews) go to serverless functions in a separate repo: **govena-backend**, deployed on Vercel at `govena-backend.vercel.app`.

The connection is handled entirely by the `_redirects` file:

```
/api/*  https://govena-backend.vercel.app/api/:splat  200
```

Netlify proxies any request to `/api/*` transparently to the Vercel backend. From the browser's perspective, the API is on the same origin — no CORS issues. **Never delete or modify this proxy rule without coordinating with the govena-backend repo.**

### API endpoints used by this site

| Endpoint | Called by | Purpose |
|----------|-----------|---------|
| `POST /api/save-answers` | `questionnaire-integration.js` (older flow) | Saves answers to KV, returns `pendingOrderId` |
| `POST /api/generate-preview` | `preview.html` | Generates tailored document excerpt via Claude |
| `POST /api/create-checkout` | `preview.html` (`goToCheckout()`) | Creates Stripe checkout session, returns redirect URL |

The Govena logo is an **inline SVG** embedded directly inside `<nav>` and `.topbar` elements in every page. It consists of:
- The Govena lettermark/icon (purple paths, `fill:#6c5ce7`)
- The word "govena" as an SVG `<text>` element using Open Sans Bold, `fill:#1e1e2f`

**Accordion FAQ:** `.faq-item` with `.faq-q` button + `.faq-body` div that transitions `max-height`. Used on `pricing.html`, `faq.html`, and `index.html`.

### Purchase flow pages

#### `before-we-begin.html` — Pre-questionnaire onboarding
**Title:** Before We Begin — Govena  
**Description:** What to expect from your Govena AI governance assessment. Takes under 10 minutes, no payment required upfront.  
Dark-themed (topbar, `#0F1729` background). Shown to customers before they start the questionnaire. Explains what information they'll need, how long it takes, and that payment comes later. Has a "Start your assessment" button linking to `questionnaire.html` and a "Back" link.

#### `questionnaire.html` — Intake questionnaire (THE CORE PAGE)
**Title:** AI Governance Pack — Intake Questionnaire | Govena  
Dark-themed app page (topbar, `#0F1729` background). 5-section multi-step form. All CSS variables use dark theme (`--color-background-primary: #1E1E2F`, etc.).

**Progress UI:** Top bar shows section counter ("Section X of 5"), percentage complete, and a step dot indicator. Back/Next buttons at the bottom.

**Sections:**
1. **Business basics** — business name (`#s1 input[type="text"]`), industry (select), employee count (radio `name="emp"`), business description (textarea)
2. **AI use** — AI tools (checkbox group `data-group="aitools"`, with "Other" free-text field `id="other-field"`), use cases (checkbox group `data-group="aiuses"`), client-facing outputs (radio `name="clientai"`), ADM use (radio `name="aiad"`)
3. **Risk and compliance** — personal info types (checkbox group `data-group="piTypes"`), personal data as AI inputs (radio `name="aidata"`), governance maturity (radio `name="mat"`), prior AI incidents (radio `name="aiinc"`)
4. **Vendors** — vendor acquisition (checkbox group `data-group="vendoracq"`), vendor review process (radio `name="vend"`), vendor location (radio `name="vendloc"`)
5. **Governance preferences** — governance owner (radio `name="owner"`), document tone (radio `name="tone"`), primary reason (radio `name="why"`)

**DOM conventions:**
- Radio options: `.radio-opt` div; class `.selected` added when chosen. Label text scraped from `.radio-text > div` (or `.radio-text` itself).
- Checkbox options: `.check-opt` div; class `.selected` toggled. Label text scraped from `.radio-text > div`.
- Groups identified by `data-group="groupname"` on the container.
- Sections: `<div id="s1">` through `<div id="s5">`. Class `active` shows the current section.

**Validation:** `REQUIRED` object maps section numbers to required fields. `validateSection(n)` returns array of missing field labels. Error shown in `#validation-msg` div.

**Summary + tier selector:** After section 5, `showSummary()` shows a summary overlay. Customer selects Pack ($499) or Consult ($1,199) tier. A `PRICES` object holds both price IDs. `selectedPrice` tracks the selection.

**Current submit flow (`proceedToPayment()`):**
1. Calls `collectAnswers()` to scrape all form inputs
2. Saves `answers` and `tier` to `sessionStorage` (`govena_answers`, `govena_tier`)
3. Redirects directly to `preview.html` — **no API call at this step**

Note: The `proceedToPayment()` function in this file does NOT call `/api/save-answers`. Answers travel to `preview.html` via `sessionStorage` only, then get saved to KV by `/api/generate-preview` (which stores them under a `previewId`). The `api/create-checkout.js` endpoint then retrieves them using that `previewId`.

#### `preview.html` — Document preview (live AI-generated)
**Title:** Your Pack Preview — Govena  
Light-themed with dark header banner. Shown immediately after the questionnaire. Makes a live call to the Claude API to generate a partial document.

**States:**
- **Loading** (`#loading-state`) — animated spinner + 4 progressive loading steps (each lights up every ~5s to simulate progress)
- **Content** (`#doc-content`) — rendered document sections with a fade-out blur at the bottom (`.doc-fade::after` gradient)
- **Error** (`#error-state`) — shown if `sessionStorage` is empty or API fails

**Init flow:**
1. Reads `govena_answers` and `govena_tier` from `sessionStorage`
2. Updates `#banner-name` with `answers.businessName`
3. Calls `POST /api/generate-preview` with `{ answers, tier }`
4. Renders the returned JSON `{ sections }` into the `#doc-body` element
5. Shows `#cta-section` with the buy button

**Document render:** The `renderSection()` function converts the JSON sections into HTML using `.doc-h1`, `.doc-h2`, `.doc-p`, `.doc-list`, `.doc-table` classes.

**Buy flow (`goToCheckout()`):**
1. Calls `POST /api/create-checkout` with `{ previewId, tier: tierSelected }`
2. Redirects to `session.url` (Stripe Checkout)

#### `success.html` — Post-payment confirmation
**Title:** Payment Successful — Govena  
Light-themed with standard nav. Shown via Stripe's `success_url` after payment. Static page — no API calls. Displays a "Payment confirmed" message and a 3-step "What happens next" timeline. Has a link back to `index.html`.

## 5. The `_redirects` file and API proxy

```
/api/*  https://govena-backend.vercel.app/api/:splat  200
```

- The `200` status code means Netlify performs a **transparent proxy** — the browser sees `govena.com.au/api/webhook` etc., not the Vercel URL
- This eliminates all CORS issues (same-origin from the browser's perspective)
- The govena-backend Vercel deployment also has CORS headers set, but they're only needed for non-Netlify callers (e.g. Stripe webhooks)
- If the Vercel backend URL ever changes, update this file and push to main to deploy

## 8. Conventions to follow when editing HTML files

### No build tools
Do not add npm, webpack, Vite, Sass, or any build tooling. The site must remain static HTML.

### All CSS in `<style>` tags
There are no external `.css` files. Keep all styles inline in `<style>` blocks in `<head>`. Do not create external stylesheets.

### Use CSS variables for colour
Every colour in the design system has a variable. Use `var(--p)` not `#6C5CE7`. If a colour doesn't have a variable, add one to `:root` before using it.

### Page title format
`Page Name — Govena` (em dash, not hyphen). Exception: homepage is just `Govena — AI Governance for Australian SMEs`.

### Every public page needs full SEO meta
Every indexable page must have:
- `<meta name="description">` — 1-2 sentences
- `<meta name="robots" content="index, follow">`
- `<link rel="canonical" href="https://govena.com.au/filename.html">`
- Full Open Graph block (type, title, description, url, image, width, height, site_name, locale)
- Twitter Card block (card, title, description, image)

Non-public pages (questionnaire, preview, success, before-we-begin, contact, sample-preview) should use `<meta name="robots" content="noindex">`.

### Never change article filenames
Articles are indexed by search engines and may have inbound links. Renaming breaks both.

### Mobile breakpoints
Every page has `@media (max-width: 900px)` for tablet/mobile. When adding new sections, always include mobile styles. Nav collapses to hamburger at 900px. Most grid layouts collapse to single column.

### Inline SVG for the logo
The logo in nav is an embedded SVG. Do not replace it with an `<img>` tag or an external SVG file reference.

### JavaScript is vanilla
No jQuery, no React, no Vue. Everything is vanilla JS. Keep it that way.

### sessionStorage for questionnaire data
Answers travel from `questionnaire.html` to `preview.html` via `sessionStorage` keys `govena_answers` (JSON string) and `govena_tier` (string: `'pack'` or `'consult'`). Do not change these key names without updating `preview.html`.

### `_redirects` must not be changed without backend coordination
The proxy URL in `_redirects` points to `govena-backend.vercel.app`. Any change to this must be tested against the backend first.

### The `govena-resources.html` design system is isolated
This file uses a completely different visual language. Do not copy CSS patterns from it into other pages.
